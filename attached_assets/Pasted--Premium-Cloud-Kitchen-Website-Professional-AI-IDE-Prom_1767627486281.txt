# Premium Cloud Kitchen Website - Professional AI IDE Prompt

## Project Brief
Develop a sophisticated, restaurant-grade website for Tushar's cloud kitchen in Delhi. This is a **professional food outlet website** first, with strategic integration of his YouTube credibility. The site must compete with premium food delivery platforms while showcasing authentic culinary expertise through his documented journey.

---

## Design Philosophy: Premium Restaurant Digital Experience

### Visual Language
- **Aesthetic**: Michelin-guide level sophistication, not casual or playful
- **Mood**: Elegant, appetizing, trustworthy, premium yet accessible
- **References**: Think Zomato Pro, Swiggy Gourmet, high-end restaurant websites
- **3D Integration**: Subtle, purposeful, enhances food presentation (NOT gimmicky or game-like)
- **Photography Style**: Professional food photography - shallow depth, warm lighting, editorial quality

### Color Psychology for Food
```
Primary Palette:
- Deep Burgundy (#7D1935) - Rich, appetite-stimulating
- Charcoal (#1A1A1A) - Sophistication, premium feel
- Cream (#FFF8E7) - Clean, fresh, appetizing backdrop
- Terracotta (#CC5500) - Warm, authentic, spice-inspired

Accent Colors:
- Gold (#D4AF37) - Premium highlights, special items
- Fresh Green (#2D5016) - Herbs, freshness, health
- Tomato Red (#C1121C) - Energy, appetite appeal

Typography:
- Headlines: Cormorant Garamond / Playfair Display (elegant serif)
- Subheadings: Montserrat SemiBold (modern, clean)
- Body: Source Sans Pro / Inter (readable, professional)
- Prices: Roboto Mono (clear, modern)
```

## 3D Implementation Guidelines - CRITICAL

### 3D Asset Quality Standards
**DO'S**:
âœ… Use photorealistic food models (high polygon count, 10k-50k vertices)
âœ… PBR materials (Physically Based Rendering): Metalness, roughness, normal maps
âœ… Realistic textures: High-res diffuse maps (2048x2048 minimum)
âœ… Professional lighting: Three-point lighting (key, fill, rim)
âœ… Subtle animations: Gentle rotation (5-10 degrees/second)
âœ… Optimize for web: Draco compression, LOD (Level of Detail)
âœ… Fallback to high-quality 2D images on low-end devices

**DON'TS**:
âŒ Cartoon-style or low-poly models
âŒ Overly bright, unrealistic colors
âŒ Fast spinning or bouncing animations
âŒ Too many 3D elements competing for attention
âŒ 3D for the sake of 3D (must serve presentation)
âŒ Heavy files causing load delays

### 3D Integration Points

**1. Hero Section** (ONE primary 3D model):
```javascript
// Example: Signature dish with subtle animation
const dish3D = {
  model: 'hero_biryani.gltf',
  position: { x: 0, y: 0, z: 0 },
  rotation: { autoRotateSpeed: 0.5 },
  lighting: {
    ambient: warm,
    directional: { intensity: 1.5, position: [5, 10, 7.5] },
    spotlight: { color: 0xffffff, intensity: 0.8 }
  },
  camera: { fov: 35, position: [0, 1, 5] },
  controls: { enableZoom: false, enablePan: false }
}
```

**2. Menu Item Modals** (On-demand loading):
```javascript
// Only load 3D when user clicks "View in 3D"
onClick: loadModel('item_burger.gltf')
- Smooth transition from 2D to 3D view
- Loading indicator: Elegant spinner
- OrbitControls: Drag to rotate, scroll to zoom
- Reset view button
```

**3. Combo Plates** (Composite 3D scenes):
```javascript
// Multiple items arranged on a plate
comboScene = {
  plate: 'base_plate.gltf',
  items: [
    { model: 'biryani_bowl.gltf', position: [-2, 0, 0] },
    { model: 'naan.gltf', position: [2, 0, 0] },
    { model: 'raita_bowl.gltf', position: [0, 0, 2] }
  ],
  presentation: 'restaurant_table_setting'
}
```

### Performance Optimization
- Lazy load 3D assets (load only when in viewport)
- Use WebGL renderer with antialias
- Implement texture compression
- Provide 2D fallback for unsupported browsers
- Max file size per model: 2-5MB
- FPS target: 60fps on mid-range devices

## Content Writing Guidelines

### Tone of Voice
- **Professional yet warm**: Not corporate, not overly casual
- **Confident**: "Fresh ingredients daily" not "We try to use fresh ingredients"
- **Descriptive**: Use sensory words (aromatic, tender, crispy, rich)
- **Authentic**: Honest about YouTube origins, don't oversell
- **Action-oriented**: Clear CTAs throughout

### Menu Description Formula
```
[Item Name]
[One-line sensory description highlighting key ingredients/preparation]

Example:
"Chicken Tikka Masala
Tender charcoal-grilled chicken in a velvety tomato cream sauce, aromatic with fenugreek and garam masala"

NOT:
"Chicken Tikka Masala
A popular Indian dish with chicken and sauce"
```

### Avoid These Phrases
âŒ "Best in Delhi" (unprovable)
âŒ "Secret recipe" (overused)
âŒ "Grandma's recipe" (unless actually true)
âŒ "Authentic" without context
âŒ Over-the-top claims
